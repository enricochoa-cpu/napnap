# 2026-03-01 — About section and app version in Support

## Summary

Added an **About** screen under Support (Profile → Support → About) that shows app name (NapNap), release version, and links to Terms of Service and Privacy policy. Terms and Privacy were removed from the Support menu so they are only reachable via About, keeping Support focused and matching common app patterns.

## Changes

### Version in build
- **vite.config.ts**: Reads `version` from `package.json` and defines `__APP_VERSION__` at build time so the UI can display the release version.
- **src/vite-env.d.ts**: Declares `__APP_VERSION__` for TypeScript (and references `vite/client`).

### About view
- **src/components/Profile/AboutView.tsx** (new): SubViewHeader, app info card (NapNap + “Version X.X.X” from `__APP_VERSION__`), and Legal section with buttons that navigate to Terms of Service and Privacy policy. Back returns to Support.

### Support and routing
- **ProfileMenu.tsx**: Added `'about'` to `ProfileView` type.
- **SupportView.tsx**: Added “About” as the first row (icon + “App version and legal” subtitle). Removed Terms of Service and Privacy policy rows; those are now only in About.
- **ProfileSection.tsx**: Import `AboutView`, `handleBackFromAbout` (back to Support), and render `AboutView` when `currentView === 'about'`. Terms/Privacy back navigation still uses `previousView`, so returning from Terms/Privacy from About goes back to About.

### i18n
- **en, es, ca**: New `support.about`, `support.aboutMenuSubtitle` and full `about` block: `title`, `subtitle`, `appName`, `version`, `legal`, `termsOfService`, `privacyPolicy`.

## Verification

- `npm run build` passes.
- Support menu: About, FAQs, Contact, then tip card. Terms and Privacy only via About → Legal.
