# 2026-04-02 — Sleep Log Enrichment (Phases 1-4)

## Changes

### Phase 1: Pause Data Layer + Edit UI
- **New table:** `sleep_pauses` (id, sleep_entry_id FK CASCADE, start_time, duration_minutes). RLS mirrors sleep_entries (owner/caregiver/viewer).
- **New type:** `SleepPause { id, sleepEntryId, startTime, durationMinutes }`
- **New utility:** `getNetSleepMinutes()` in dateUtils — gross duration minus pause minutes. Used in ALL duration displays (TodayView, SleepEntry, SleepList, DailySummary, StatsView, reportData).
- **Hook:** `useSleepEntries` fetches pauses in batch (`.in('sleep_entry_id', ids)`), merges into entries. New CRUD: `addPause`, `updatePause`, `deletePause` with optimistic local state updates.
- **SleepEntrySheet:** Pause section with collapsible cards (Pause N — start · duration), expand to edit start time + duration. "+ Add pause" dashed button (max 5). Duration input uses `defaultValue` + `onBlur` to allow multi-digit typing.
- **Bug fix:** `editingEntry` was a stale snapshot → changed to `editingEntryId` + `useMemo` deriving from live `entries` array.
- **Bug fix:** `useEffect` reset `expandedPauseId` on every entry reference change → scoped to `entryId` identity.
- **Bug fix:** Cross-midnight pause start time used wrong date → detects post-midnight with `isTimeBefore` + `nextDayStr`.

### Phase 2: Live Pause/Resume
- **App.tsx:** `activePauseStart: Date | null` state, passed to SleepEntrySheet + TodayView.
- **SleepEntrySheet:** Two buttons for active entries: Pause/Play (left, pulsing ring when paused) + Stop (right). `handlePause` sets start time, `handleResume` computes duration and creates pause record. `handleSave` ends in-flight pause before stopping. Duration label shows net elapsed (gross − completed pauses − in-flight).
- **TodayView:** `currentSleepDuration` accounts for completed + in-flight pauses. "Paused" badge on hero card and active nap timeline card.
- **Sheet close while paused:** Pause discarded (intentional safety).
- **Bug fix:** Pause section only showed for completed entries → changed to also show for active entries with pauses.

### Phase 3: Night Waking
- **Contextual labels:** When `entry.type === 'night'`, pauses show as "Night waking N" with storm cloud icon and `--wake-color` text. "+ Add night waking" instead of "+ Add pause". "Night waking" instead of "Paused" for live pause status.
- **QuickActionSheet:** "Night waking" button in second row (storm cloud, `--wake-color`). Only visible when `hasNightEntry` (active or recent bedtime). Tap: active bedtime → starts live pause; completed → opens entry for retrospective pause.
- **DailySummary:** `nightWakingCount` and `nightWakingMinutes` in `computeDailySummary`. "X night wakings (Ym total)" row with storm cloud icon when count > 0.

### Phase 4: Qualitative Information
- **New columns:** `onset_tags text[]` and `sleep_method text` on `sleep_entries`.
- **Types:** `onsetTags?: string[]` and `sleepMethod?: string` on SleepEntry/DbSleepEntry.
- **Hook:** Maps new fields in fetch, add, update.
- **SleepEntrySheet:** Onset chips (multi-select: Long time to fall asleep, Upset) + method chips (single-select: In bed, Nursing, Worn or held, Next to me, Bottle feeding, Stroller, Car, Swing) + notes textarea. All optional.
- **i18n:** All chip labels in en/es/ca.

## Migrations
- `20260402000000_sleep_pauses.sql` — sleep_pauses table + RLS
- `20260402000001_sleep_qualitative.sql` — onset_tags + sleep_method columns

## Decisions
- Net sleep replaces gross duration everywhere — parents care about actual sleep, not time in bed.
- Night waking = pause on night entry (same data, different label). No separate data model.
- Qualitative tags are optional, never required — aligns with PRD "no judgement" principle.
- Live pause state is local (not persisted) — lost on app refresh. Acceptable for MVP.
- Report integration for qualitative data deferred to backlog.
