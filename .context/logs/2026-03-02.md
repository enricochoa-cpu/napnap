# 2026-03-02 — Growth chart Y-axis overlap fix & trends UI tweaks

## Summary

Fixed the persistent overlap in Trends → Growth: the "2 kg" Y-axis label no longer overlaps the first X-axis label ("dl. 16/6"). Also tightened legend spacing on Daily Sleep and documented the fix.

## Changes

### Growth charts (Weight / Height over time)
- **Root cause:** Recharts draws the lowest Y tick and the first X tick in the same bottom-left corner. Increasing margins alone did not fix it; the domain could still include a tick at 2 kg when data started at ~4 kg.
- **Fix:** Filter Y-axis ticks by actual data range:
  - `weightTicksFiltered`: only ticks ≥ min(weightLogs) − 0.5 (drops "2 kg" when curve starts at 4 kg).
  - `heightTicksFiltered`: only ticks ≥ min(heightLogs) − 5.
- **StatsView.tsx:** Added `weightTicksFiltered` and `heightTicksFiltered` useMemos; all growth AreaCharts use these for the YAxis `ticks` prop. Fallback to unfiltered ticks if filtered result is empty.

### Margins and legend spacing (from earlier audit)
- **CHART_MARGIN_GROWTH:** `left: 48`, `bottom: 48` (was 40, 44) so Y labels and two-line X labels have room.
- **Son diari (Daily Sleep):** Legend row spacing reduced from `mt-3` to `mt-1` (all three instances: distribution, Summary section, Naps section).

### Documentation
- **lessons.md §7.6:** New entry for growth chart Y-tick overlap (margin not enough; permanent fix = filter ticks by data min).
