### Landing Page Redesign (LandingPage.tsx, index.css)

**7-step overhaul** to improve conversion and visual polish:

1. **Nav fix**: Scroll-aware solid background after 200px (`.glass-nav-landing--solid`). Prevents content peeking through transparent nav.
2. **Social proof**: Replaced weak star-rating grid with **cinematic testimonial carousel** — auto-rotate every 6s, touch swipe support, dot indicators (56px touch targets), warm `--bg-mid` background band with decorative open-quote SVG.
3. **Product showcase (NEW)**: "See it in action" section with 3 device-framed screenshots (Today/NAP NOW, Sleep Log, Trends). Horizontal snap-scroll on mobile, 3-col grid on desktop. Uses existing `.device-frame` CSS classes.
4. **Mid-page CTA (NEW)**: "Ready to find your rhythm?" + reassurance text + primary CTA button.
5. **Section consolidation**: Removed Sleep Guides (~50 lines) and Sleep Science (~35 lines) sections. Lightened Age Range (removed card wrapper and PairedCyclingVideos, kept heading+text+tags). Folded trust messaging into How It Works subtitle.
6. **Visual rhythm breaks**: Wrapped "What you get" in `--bg-mid` tinted band (full-bleed mobile, rounded desktop).
7. **Cleanup**: Removed dead `StarIcon`, `PairedCyclingVideos`, `PAIRED_VIDEO_CLASS`. Added "How much does NapNap cost?" FAQ. Nav link "Sleep science" → "The app" pointing to `#product-showcase`.

**Screenshots**: Extracted user-provided app screenshots (Today/NAP NOW, Sleep Log, Trends, Baby Profile) from conversation, saved to `public/media/napnap-today-new.png`, `napnap-sleeplog.png`, `napnap-trends.png`, `napnap-profile.png`.

### Waitlist Email Notification (NEW Edge Function)

- **Edge Function**: `supabase/functions/waitlist-notify/index.ts` — receives email from landing page, sends styled notification to `getnapnap@gmail.com` via Resend (already configured).
- **Frontend**: `handleEmailSubmit` now calls `supabase.functions.invoke('waitlist-notify', { body: { email } })` with loading state, error handling, and input validation.
- **Config**: Added `[functions.waitlist-notify] verify_jwt = false` to `supabase/config.toml`.
- **Deploy**: Requires `npx supabase functions deploy waitlist-notify --no-verify-jwt`.

### Profile Creation Fix (useBabyProfile.ts)

- **Bug**: Creating a baby profile failed with `23505` (duplicate key) when a profile row already existed for the user.
- **Root cause**: `createProfile` used `.insert()` which fails on existing rows.
- **Fix**: Changed to `.upsert(insertPayload, { onConflict: 'id' })`, matching the pattern already used in `updateProfile`.

## Decisions

- **Testimonial carousel over static grid**: Auto-rotating carousel with swipe creates cinematic feel; static grids are too "template-y" for the brand.

- **Upsert over insert+update branching**: Simpler, idempotent, matches existing `updateProfile` pattern.
